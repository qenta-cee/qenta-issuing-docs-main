Google Pay™

Google Pay™

The Google Pay API enables fast, simple checkout on the website, and gives the merchant convenient access to hundreds of millions of cards saved to Google Accounts worldwide.

Registration

Before Google Pay API implementation is production-ready, register with Google, accept the Terms of Service, and receive a Google merchant ID after the website passes a Google review.

Before applying for production access check the following:

  • A Google Account

  • A supported payments gateway or a Payment Card Industry Data Security Standard (PCI DSS) compliant server environment that’s suitable to store decrypted payment information

  • A list of fully qualified domains, which includes subdomains, from which the merchant intend to call the Google Pay API

After the website is approved, the merchant will receive a Google merchant ID that must be included with each Google Pay API request.

Once merchant ID is generated it should be forwarded to the integration specialist.

Parameters

Google Pay API payment method can be initialized if the merchant sends GOOGLEPAY for parameter paymentType.

allowedCardNetworks

MERCHANTS can set the ccard brand for google pay and allowedCardNetworks in checkbox should be chosen:

allowedCardNetworks=AMEX,DISCOVER,DINERS,MASTERCARD,VISA

Possible value(s) of allowedCardNetworks Description

AMEX

American Express

DISCOVER

Discover

DINERS

Diners Club

MASTERCARD

Mastercard

VISA

Visa

By default all brands will be activated.

buttonType

Merchants can configure buttonType and the example is here.

Type Description

book

The "Book with Google Pay" payment button.

buy

The "Buy with Google Pay" payment button (default).

checkout

The "Checkout with Google Pay" payment button.

donate

The "Donate with Google Pay" payment button.

order

The "Order with Google Pay" payment button.

pay

The "Pay with Google Pay" payment button.

plain

The Google Pay payment button without the additional text.

subscribe

The "Subscribe with Google Pay" payment button.

buttonLocale

The ISO 639-1 code represents the desired button language.

Supported locales include: en, ar, bg, ca, cs, da, de, el, es, et, fi, fr, hr, id, it, ja, ko, ms, nl, no, pl, pt, ru, sk, sl, sr, sv, th, tr, uk, and zh.

All languages are optional and the default is what is set in the language settings of the browser or operating system.

General information about the integration

QENTA integrated for web merchants based on the following documentations:

  • Google Pay Web developer documentation

  • Google Pay Web integration checklist

  • Google Pay Web Brand Guidelines

There is no need to follow these integration guidelines as they are already covered within the QENTA products.

Information about the authorization methods can be found here.

QENTA offers in the hosted payment solutions, which allows the merchant to communicate directly with the QENTA API.

encrypted payment information (cryptogram) is handled directly by QENTA.

In general, gateway credentials are mandatory for the integration. The gateway and gatewayMerchantId are set by QENTA and are not needed to be set by the merchant.
